Use Case Diagram - Tổng quan hệ thống

Mô tả: Mô tả tất cả các chức năng của hệ thống FoodStreetGuide từ góc nhìn Actor (User và Admin). Bao gồm 12 use cases cho Mobile App, 7 use cases cho Web Admin, và 5 use cases cho System.

🎭 Các Actor tham gia:

UC7: Viết đánh giá Review và rating quán ăn include Sync
UC9: Quét QR check-in Scan QR để nhanh vào chi tiết POI extend POI Detail

Activity Diagram - App Launch & Khởi động

Mô tả: Luồng hoạt động khi người dùng mở app từ lúc bấm icon cho đến khi hiển thị bản đồ sẵn sàng sử dụng. Bao gồm kiểm tra permission, khởi tạo services, và load dữ liệu.

📝 Các bước chính:

Bước Hành động Kết quả
1 User mở app Trigger OnStart
2 Init SQLite Database Tables ready
3 Check GPS Permission Granted / Denied
4 Init GeofenceEngine Engine enabled
5 Test API Connection Online / Offline
6 Load & Display Map App ready
🔄

Activity Diagram - Geofence Trigger & Auto Narration

Mô tả: Luồng xử lý khi user di chuyển vào vùng geofence của một POI. Hệ thống tính toán khoảng cách, kiểm tra priority, cooldown, và quyết định có phát audio hay không.

🎯 Quy tắc ưu tiên:

Ưu tiên Tiêu chí Ví dụ
1 Priority cao hơn P3 > P2 > P1
2 Khoảng cách gần hơn 50m > 80m
3 Cooldown đã hết 5 phút
📍

Activity Diagram - User Review Flow

Mô tả: Luồng người dùng viết và gửi đánh giá cho một POI. Bao gồm các bước: mở form, nhập rating/comment, kiểm tra spam, lưu local, đồng bộ lên server, và chờ admin duyệt.

📝 Các trạng thái review:

Trạng thái Mô tả Màu
pending Chờ admin duyệt 🟡 Vàng
approved Đã duyệt, hiển thị public 🟢 Xanh
rejected Bị từ chối (spam/inappropriate) 🔴 Đỏ
⭐

Activity Diagram - Data Synchronization

Mô tả: Luồng đồng bộ dữ liệu giữa Mobile App (SQLite) và Web Admin (MySQL/JSON). Xử lý các trường hợp: có mạng, không mạng, lỗi server, conflict resolution.

🔄 Chiến lược đồng bộ:

Tình huống Hành động
Online + Remote mới hơn Update local database
Online + Local mới hơn Push lên server
Offline Queue để sync sau
Lỗi 500 Retry 3 lần
🔄

Activity Diagram - QR Code Scan & Check-in

Mô tả: Luồng user quét QR code tại quán để nhanh chóng vào trang chi tiết POI và check-in. Bao gồm xử lý camera permission, parse QR content, validate POI ID.

📱 QR Content Format:

Loại QR Nội dung Kết quả
Deep Link `foodstreetguide://poi/123` Mở app → POI #123
Web URL `https://.../poi/123` Mở app → POI #123
Direct ID `123` Mở app → POI #123
Invalid Không đúng format Báo lỗi
📱

Hướng dẫn tạo hình ảnh từ Mermaid

Cách 1: Mermaid Live Editor (Online)

1. Truy cập: https://mermaid.live/
2. Copy code từ box màu đen bên trên
3. Paste vào editor
4. Click "PNG" hoặc "SVG" để tải về
5. Chèn vào placeholder trong file này

Cách 2: PlantUML (Local)

1. Cài đặt PlantUML extension cho VS Code
2. Tạo file .puml với cú pháp PlantUML
3. Alt + D để preview
4. Export PNG/SVG

Cách 3: draw.io / diagrams.net

1. Truy cập: https://app.diagrams.net/
2. Chọn template Use Case / Activity
3. Vẽ bằng kéo thả
4. File → Export as → PNG/SVG/PDF
