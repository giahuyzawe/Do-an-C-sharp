App Launch & Initialization

Mô tả: Khởi động app, init services, database. Mô tả quá trình khởi tạo ứng dụng từ khi user mở app cho đến khi hiển thị bản đồ với các điểm POI.

🎭 Thành phần tham gia:

Thành phần Vai trò
User Người dùng mở ứng dụng
App FoodStreetGuide App - điều phối chính
DatabaseService Khởi tạo SQLite, tạo bảng, seed data
GeofenceEngine Khởi tạo engine theo dõi vị trí
LocationService Kiểm tra quyền GPS
TTSService Khởi tạo Text-to-Speech
WebAdminService Test kết nối API

📝 Các bước chính:

  1. User mở App
  2. Init SQLite Database (Create tables: POI, Review)
  3. Seed test data nếu database trống
  4. Khởi tạo GeofenceEngine
  5. Kiểm tra GPS permission
  6. Initialize TTS engine
  7. Test connection đến Web Admin API
  8. Load POIs từ database local
  9. Hiển thị bản đồ với markers
  10. App ready cho user

    Web-->>App: Connection status
    
    App->>App: Load POIs from database
    App->>App: Display map with markers
    App-->>User: App ready

Geofence Trigger - Auto Narration

Mô tả: Auto narration khi vào vùng POI. Mô tả quá trình theo dõi vị trí người dùng, phát hiện khi vào vùng geofence của một POI và tự động phát thông tin audio.

🎭 Thành phần tham gia:

Thành phần Vai trò
User Di chuyển gần POI
LocationService Cập nhật vị trí GPS
GeofenceEngine Tính khoảng cách, kiểm tra geofence
MainPage Hiển thị UI, điều phối audio
AudioPlayerService Phát thông báo âm thanh
TTSService Text-to-Speech narration

📝 Các bước chính:

  1. User di chuyển gần POI
  2. GPS Update (5-30s interval)
  3. Tính khoảng cách đến tất cả POIs
  4. Kiểm tra POI nào trong bán kính
  5. Sort theo Priority (3→1) rồi Distance
  6. Kiểm tra cooldown (5 phút)
  7. Nếu pass cooldown → POIEntered event
  8. Show POI Card
  9. Play audio notification
  10. Auto-narration (TTS) nếu enabled

Hình 2: Geofence Trigger

sequenceDiagram
    actor User
    participant Loc as LocationService
    participant Geo as GeofenceEngine
    participant Main as MainPage
    participant Audio as AudioPlayerService
    participant TTS as TTSService

    User->>User: Walking near POI
    
    loop GPS Update (5-30s interval)
        Loc->>Loc: Get current location
        Loc->>Main: OnLocationUpdated(lat, lng)
        Main->>Geo: UpdateLocation(lat, lng)
        
        Geo->>Geo: Calculate distance to all POIs
        Geo->>Geo: Check which POIs are inside radius
        
        alt Inside geofence & Priority check
            Geo->>Geo: Sort by Priority (3→1) then Distance
            Geo->>Geo: Check cooldown (5 min)
            
            alt Not in cooldown
                Geo->>Main: POIEntered event
                Main->>Main: Show POI Card
                Main->>Audio: PlayAudioNotification()
                
                alt Auto-narration enabled
                    Main->>TTS: SpeakAsync(description)
                    TTS-->>Main: Speaking...
                end
                
                Main-->>User: Display POI info + audio playing
            else In cooldown
                Geo->>Geo: Log "SKIP - cooldown active"
            end
        end
    end

Data Synchronization (Web → Mobile)

Mô tả: Đồng bộ dữ liệu POI từ Web Admin xuống Mobile App. Mô tả quá trình lấy dữ liệu mới nhất từ server và cập nhật vào database local.

🎭 Thành phần tham gia:

Thành phần Vai trò
User Yêu cầu đồng bộ dữ liệu
App FoodStreetGuide App
DatabaseService SQLite local CRUD
WebAdminService Gọi API và xử lý dữ liệu
Web Admin API REST API (PHP)
MySQL Database Database server

📝 Các bước chính:

  1. User tap "Sync from Web"
  2. App gọi SyncFromWebAdminAsync()
  3. HTTP GET /api/pois.php
  4. MySQL SELECT * FROM pois
  5. Trả về List of POIs (JSON)
  6. Loop qua từng POI:
  7. → Kiểm tra POI đã tồn tại local?
  8. → Nếu chưa có: INSERT mới
  9. → Nếu có: Compare LastSyncFromWeb
  10. → Nếu remote mới hơn: UPDATE
  11. Tương tự cho Reviews
  12. Sync completed → Refresh map

Hình 3: Data Synchronization

sequenceDiagram
    participant User
    participant App as FoodStreetGuide App
    participant DB as DatabaseService
    participant WebSvc as WebAdminService
    participant API as Web Admin API
    participant MySQL as MySQL Database

    User->>App: Tap "Sync from Web"
    
    App->>WebSvc: SyncFromWebAdminAsync()
    WebSvc->>API: GET /api/pois.php
    API->>MySQL: SELECT * FROM pois
    MySQL-->>API: List of POIs
    API-->>WebSvc: JSON response
    
    loop Each POI from Web
        WebSvc->>DB: GetPOIsAsync() - check existing
        DB-->>WebSvc: Local POI or null
        
        alt POI not exists locally
            WebSvc->>DB: AddPOIAsync(newPOI)
            DB->>DB: INSERT INTO POI
            DB-->>WebSvc: Success
        else POI exists - check update time
            WebSvc->>WebSvc: Compare LastSyncFromWeb
            alt Remote is newer
                WebSvc->>DB: UpdatePOIAsync(poi)
                DB->>DB: UPDATE POI
            end
        end
    end
    
    WebSvc->>DB: Same for Reviews
    
    WebSvc-->>App: Sync completed
    App->>App: Refresh map markers
    App-->>User: Show sync success message

User Reviews Flow

Mô tả: Tạo và đồng bộ đánh giá. Mô tả quá trình user viết review, lưu local, và đồng bộ lên server khi có network.

🎭 Thành phần tham gia:

Thành phần Vai trò
User Viết và submit review
POIDetailPage Form review, xử lý submit
DatabaseService Lưu review local
WebAdminService Đồng bộ review lên server
Web Admin API Nhận và lưu review

📝 Các bước chính:

  1. User xem POI details
  2. Load danh sách reviews (approved)
  3. User tap "Write Review"
  4. Hiển thị review form
  5. User nhập rating + comment + images
  6. Submit review
  7. Lưu vào SQLite local
  8. Nếu có network: POST /api/reviews.php
  9. Nếu offline: Queue for later sync
  10. Thông báo "Review submitted"

Hình 4: User Reviews Flow

    Detail-->>User: Display reviews

    User->>Detail: Tap "Write Review"

    User->>Detail: Enter rating + comment + images
    User->>Detail: Submit review
    
    Detail->>DB: AddReviewAsync(review)

    DB-->>Detail: Review saved locally
    
    alt Network available
        Detail->>Web: POST /api/reviews.php
        Web->>API: Forward review data
        API->>MySQL: INSERT INTO reviews
        MySQL-->>API: Success
        API-->>Web: Review created
        Web-->>Detail: Sync success

    else Offline
        Detail->>Detail: Queue for later sync
    end
    
    Detail-->>User: Show "Review submitted"

LocationService Theo dõi GPS background
GeofenceEngine Kiểm tra geofence
NotificationService Hiển thị thông báo
AudioService Phát audio khi vào geofence

📝 Các bước chính:

  1. User tap "Start Tracking"
  2. Yêu cầu background location permission

QR Code Scan & Check-in

Mô tả: Quét QR để check-in. Mô tả quá trình quét mã QR tại quán để nhanh chóng xem thông tin và check-in.

🎭 Thành phần tham gia:

Thành phần Vai trò
User Quét QR code
QRScanPage UI quét QR
CameraService Điều khiển camera
QRDecoder Giải mã QR
DatabaseService Lấy thông tin POI
GeofenceEngine Kiểm tra khoảng cách (optional)

📝 Các bước chính:

  1. User tap QR Scan button
  2. Kiểm tra camera permission
  3. Start camera preview
  4. User chĩa camera vào QR code
  5. Capture frame → Decode QR
  6. Extract POI ID
  7. Lấy POI details từ database
  8. Validate proximity (optional)
  9. Nếu hợp lệ: Increment visit count
  10. Show "Check-in successful!"

Hình 7: QR Code Scan

sequenceDiagram
    actor User
    participant QR as QRScanPage
    participant Camera as CameraService
    participant Decoder as QRDecoder
    participant DB as DatabaseService
    participant Geo as GeofenceEngine

    User->>QR: Tap QR Scan button
    QR->>QR: Check camera permission
    QR->>Camera: Start camera preview
    Camera-->>QR: Camera active
    QR-->>User: Show camera view

    User->>User: Point camera at QR code
    Camera->>Decoder: Capture frame
    Decoder->>Decoder: Decode QR content
    Decoder-->>Camera: POI ID extracted

    Camera->>QR: OnQRCodeDetected(poiId)
    QR->>DB: GetPOIAsync(poiId)
    DB-->>QR: POI details

    alt Valid POI
        QR->>Geo: Validate proximity (optional)
        Geo->>QR: Distance check result
        
        alt Within range or no check
            QR->>DB: IncrementVisitCount(poiId)
            QR->>QR: Show POI details
            QR-->>User: "Check-in successful!"
        else Too far
            QR-->>User: "Please move closer"
        end
    else Invalid POI
        QR-->>User: "Invalid QR code"
    end

    User->>QR: Tap close/done
    QR->>Camera: Stop camera
    QR->>QR: Navigate back

Audio Narration Pipeline

Mô tả: TTS và audio playback. Mô tả quá trình phát audio thuyết minh, bao gồm cả pre-recorded và TTS real-time.

🎭 Thành phần tham gia:

Thành phần Vai trò
User Nhấn Listen/Stop
MainPage Điều phối audio
TTSService Text-to-Speech
AudioPlayerService Phát audio
Platform Audio Native audio API

📝 Các bước chính:

  1. User tap "Listen" button
  2. Get POI description
  3. Nếu có pre-recorded audio:
  4. → PlayFromUrl(audioUrl)
  5. → Stream audio
  6. Nếu không có: Use TTS
  7. → Split text into sentences
  8. → Loop: TTS each sentence
  9. → Synthesize speech
  10. → Play audio chunk
  11. User tap "Stop" → Cancel TTS + Stop playback

Hình 9: Audio Narration Pipeline

sequenceDiagram
    actor User
    participant Main as MainPage
    participant TTS as TTSService
    participant Audio as AudioPlayerService
    participant Platform as Platform Audio

    User->>Main: Tap "Listen" button
    Main->>Main: Get POI description
    
    alt Pre-recorded audio exists
        Main->>Audio: PlayFromUrl(audioUrl)
        Audio->>Platform: Initialize MediaPlayer
        Platform->>Platform: Stream audio
        Platform-->>Audio: Playing...
        Audio-->>Main: Playback started
    else Use TTS
        Main->>TTS: SpeakAsync(description, lang)
        TTS->>TTS: Split text into sentences
        
        loop Each sentence
            TTS->>Platform: TextToSpeech(sentence)
            Platform->>Platform: Synthesize speech
            Platform-->>TTS: Audio data
            TTS->>Audio: Play audio chunk
            Audio->>Platform: Output to speaker
        end
        
        TTS-->>Main: TTS completed
    end

    User->>Main: Tap "Stop"
    Main->>Audio: Stop()
    Audio->>Platform: Stop playback
    Audio->>TTS: Cancel()
    Platform-->>Audio: Stopped
    Audio-->>Main: Audio stopped

Error Handling & Recovery

Mô tả: Xử lý lỗi network, database. Mô tả cách app xử lý các lỗi network, API error, database corrupt và recovery.

🎭 Thành phần tham gia:

Thành phần Vai trò
App FoodStreetGuide App
WebAdminService Gọi API
DatabaseService SQLite operations
Debug Logger Ghi log lỗi

📝 Các bước chính:

  1. App gọi SyncFromWebAdminAsync()
  2. HTTP GET request
  3. Network error:
  4. → Log error details
  5. → Return cached data
  6. → Get local POIs
  7. → Show offline indicator
  8. → Queue sync for retry
  9. API error (500):
  10. → Log API error
  11. → Show error message + Retry button
  12. Database corrupt:
  13. → Log crash info
  14. → Reset database
  15. → Re-initialize + Force full sync

Hình 10 :Error Handling

sequenceDiagram
    participant App as FoodStreetGuide App
    participant Web as WebAdminService
    participant DB as DatabaseService
    participant Logger as Debug Logger

    App->>Web: SyncFromWebAdminAsync()
    Web->>Web: HTTP GET request
    
    alt Network error
        Web->>Logger: Log error details
        Web-->>App: Return cached data
        App->>DB: GetLocalPOIsAsync()
        DB-->>App: Local POIs
        App->>App: Show offline indicator
        App->>App: Queue sync for retry
    else API error (500)
        Web->>Logger: Log API error
        Web-->>App: Error response
        App->>App: Show error message
        App->>App: Offer retry button
    else Success
        Web-->>App: Fresh data
        App->>DB: Update local database
    end

    alt Database locked/corrupt
        App->>DB: Query POIs
        DB->>DB: SQLite exception
        DB-->>App: Throw exception
        App->>Logger: Log crash info
        App->>App: Reset database
        App->>DB: Re-initialize
        DB-->>App: Fresh database
        App->>Web: Force full sync
    end

📚 Chú thích ký hiệu Sequence Diagram

Ký hiệu Ý nghĩa
->> Synchronous call (gọi đồng bộ)
--> Return response (trả về kết quả)
-> Asynchronous event (sự kiện bất đồng bộ)
alt Alternative flow (rẽ nhánh if/else)
loop Repeated operation (vòng lặp)
par Parallel processing (xử lý song song)
activate Object active (đối tượng đang hoạt động)
deactivate Object inactive (đối tượng ngừng hoạt động)
